“第一篇博客文章”
Created 2020-02-21| Updated 2020-02-21
Post View:

绪论

我叫帅大谦
这是我第一次博客搭建
寒假科协小任务之一
下面介绍一下这次博客搭建的过程和踩的坑

经验

寒假过了一半才真正开始接触这东西
在科协的时候感觉他们都是大佬，觉得离他们很遥远，离程序员很远
一直静不下心来认认真真的学，因为这次的任务我才开始看有关这些的博文和视频

然后在2020.02.21凌晨0：00开始跟着博主一步一步搭建博客

安装node.js—>用cmd命令行利用镜像源安装cnpm—>用cnpm全局安装hexo框架—>
初始化框架！！！—>完成在本地搭建—>生成新的博客文章（markdown语法）—>在github上创建新仓库！！！—>安装git的一个部署的插件hexo-deployer-git—>设置_config.yml中的#Deployment！！！—>部署到远端上！！！（以后用GitHub来访问博客）

至此，博客搭建完成，之后会完善博客（小声bb：应该吧）

1.初始化框架：在这一步发现电脑里还没安装git（坑一反反复复试了好几次，查询各种网站才知道windows要安装配置git才能初始化hexo框架，然后在blog文件夹中用git bash中hexo init即可初始化框架）
2.仓库名称：仓库的名字必须是 昵称.github.io 也是以后博客的访问地址
3.设置#Deployment：冒号之后要空格！！保存退出
deploy:
type: ‘git’（单引号不能省）
repo: https://github.com/shuaidaqian/shuaidaqian.github.io.git
banch: master
4.推到远端：git没有配置好，设置好用户名和邮箱地址，配置ssh公钥！
在Git Bash Here下输入ssh-keygen -t rsa -C “邮箱地址” 命令,在C:\Users\计算机名称.ssh文件下用记事本方式打开public文件，复制密钥，在GitHub的settings下找到SSH and GPG keys，选择New SSH key，回到Git Bash Here重新部署hexo d，重新部署前把.deployer_git先删掉然后 hexo clean清理 hexo g重新生成 hexo d部署到远端
